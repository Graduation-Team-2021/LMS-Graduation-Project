USER

Concerns with users both students and professors.
  • SIGN UP

    POST /sign_up

    Creates a new professor or a student based on the role attribute and returns an auth token.
    Request for professors: 
    
                            
                            {
                                    "name":"professor name",
                                    "email":"email",
                                    "national_id":"4837182373481",
                                    "birthday":"1996-10-10",
                                    "password":"password",
                                    "scientific_degree":"PHD",
                                    "role":"professor"   
                                }
    Request for students: 
    
                            {
                                "name":"student name",
                                "email":"email",
                                "national_id":"4837182373481",
                                "birthday":"1996-10-10",
                                "password":"password",
                                "student_year":4,
                                "role":"student"   
                            }
  • LOG IN

    POST /login

    Logs the user with their email and password and returns an auth token.
    Request example: 
    
                            {
                                "email":"email",
                                "password":"password"
                            }
  • EDIT USER

    POST /users/<user_id>

    Request for professors: 
    
                            
                            {
                                "name":"professor name",
                                "email":"email",
                                "national_id":"4837182373481",
                                "birthday":"1996-10-10",
                                "scientific_degree":"PHD", 
                            }
    Request for students: 
    
                            {
                                "name":"student name",
                                "email":"email",
                                "national_id":"4837182373481",
                                "birthday":"1996-10-10",
                                "student_year":4,   
                            }
  • DELETE USER

    DELETE /users/<user_id>

  • GET USER INFORMATION

    GET /users/<user_id>

    Response example: 
    
                            {
                                "user_id": 45,
                                "name": "name",
                                "email": "email",
                                "national_id":"43365126839",
                                "birthday":"birthday"
                            }

COURSE

  • ADD COURSE

    POST /courses

    Request example: 
    
                            {
                                "course_code":1,
                                "course_name":"course_name",
                                "weekly_hours":3,
                                "group_number":2,
                                "max_students":10
                            }
  • GET COURSE

    GET /courses/<course_code>

    Response example: 
    
                            "course":{
                                'course_code': "CS452",
                                'course_name': "Control Systems",
                                'weekly_hours': 5,
                                'group_number': 10,'max_students': 100
                                },
                                "status_code":200

    PUT /courses/<course_code>

    Request example: 
    
                            "course":{
                                'course_code': "CS452",
                                'course_name': "Control Systems",

  • UPLOAD DELIVERABLE

    To upload a deliverable two steps needs to happen:

    1. POST /my_deliverables (Requires auth token):

    Request example: 
    
                            {
                                "deliverable_id":1
                            }

    A delivers relation id is then returned and used to upload the file.

    Request response: 
    
                        {
                            "delivers_id": 5,
                            "message": "delivers relation added successfully",
                            "status_code": 200
                        }

    2. POST /my_deliverables/<delivers_id>/upload (Requires auth token):

    Request example: 
    
                            file: assignment.pdf
  • DOWNLOAD DELIVERABLE

    POST /my_deliverables/<delivers_id>/download

  • DELETE STUDENTS SUBMISSION
